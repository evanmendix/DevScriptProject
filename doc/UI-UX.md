# UI/UX 設計文件

本文件描述了應用程式的使用者介面 (UI) 佈局與使用者體驗 (UX) 流程。

---

### 1. 主視窗設計

應用程式的主視窗採用現代化的兩欄式佈局，旨在提供清晰、高效的操作流程。

- **左側面板 (管理區)**:
    - **搜尋與篩選**: 頂部是一個控制區，包含一個**搜尋框**和一組**類型篩選按鈕** (`ALL`, `PY`, `BAT`, `PS1`)。這些控制項讓使用者能快速縮小腳本列表的範圍。
    - **腳本列表**: 一個可滾動的列表，顯示所有（經過篩選後）的腳本。每個腳本都是一個可點擊的按鈕。列表頂部有一個「**＋ 新增腳本**」的按鈕。

- **右側面板 (互動區)**:
    - **標題**: 顯示當前所選腳本的名稱。
    - **主控台輸出**: 一個大的文字區域，用於即時顯示腳本執行時的 `stdout` 和 `stderr` 輸出。
    - **控制按鈕區**: 底部是一排按鈕，會根據所選腳本的狀態動態變化。

---

### 2. 核心使用者流程 (User Flows)

#### 2.1 新增一個腳本
1.  使用者點擊左上角的「**＋ 新增腳本**」按鈕。
2.  彈出一個「新增腳本」的對話框視窗。
3.  使用者點擊「**瀏覽...**」按鈕，開啟系統檔案總管來選擇一個 `.py`, `.bat`, 或 `.ps1` 檔案。
4.  選擇檔案後，路徑會被自動填入，腳本名稱也會被預設為檔案的主檔名。
5.  使用者可以自訂「**顯示名稱**」。
6.  使用者從下拉選單中選擇腳本類型：「**一次性**」或「**背景常駐**」。
7.  點擊「**儲存**」，對話框關閉，新的腳本出現在左側的列表中。

#### 2.2 執行一個腳本
1.  使用者從左側列表中點擊一個腳本。
2.  右側面板的標題更新為該腳本的名稱，主控台區域清空並顯示提示訊息。
3.  底部的控制按鈕區根據腳本類型更新：
    - **一次性腳本**: 顯示「**執行**」按鈕。
    - **背景腳本**: 顯示綠色的「**啟動**」按鈕。
4.  使用者點擊按鈕。
5.  腳本開始執行，其輸出會即時地顯示在主控台區域。
6.  對於背景腳本，「啟動」按鈕會變為紅色的「**停止**」按鈕。

#### 2.3 編輯一個腳本
1.  使用者從左側列表中點擊一個腳本。
2.  點擊右側控制區的「**編輯**」按鈕。
3.  彈出「編輯腳本」對話框，所有欄位都已預先填入該腳本的現有資料。
4.  使用者修改名稱或類型後，點擊「**儲存**」。
5.  左側列表和右側標題的名稱會即時更新。

#### 2.4 移除一個腳本
1.  使用者從左側列表中點擊一個腳本。
2.  點擊右側控制區最右邊的紅色「**移除**」按鈕。
3.  彈出一個確認對話框，詢問使用者是否確定要移除。
4.  使用者點擊「**確定移除**」。
5.  該腳本從左側列表中消失，右側面板恢復到未選擇狀態。
