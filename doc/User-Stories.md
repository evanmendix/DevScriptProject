# 使用者故事 (User Stories)

本文件描述了此腳本啟動器應用程式的目標使用者及其需求。

---

### 核心使用者

- **角色**: 開發人員、系統管理員、或任何需要頻繁執行本地腳本的進階使用者。
- **背景**: 使用者擁有多個 `.bat`, `.ps1`, 或 `.py` 腳本，散佈在不同的專案目錄中。他們需要一個比手動開啟終端機更有效率、更視覺化的方式來管理和執行這些腳本。

---

### 主要功能故事

1.  **作為一名使用者，我想要一個集中的介面來管理我所有的腳本，這樣我就不必在檔案總管中到處尋找它們。**
    - **驗收條件**:
        - 應用程式提供一個列表，顯示所有已加入的腳本。
        - 我可以透過「新增腳本」按鈕，從檔案系統的任何位置加入新的腳本。

2.  **作為一名使用者，我想要能為我的腳本取一個好記的名稱，因為檔案名稱本身可能不夠直觀。**
    - **驗收條件**:
        - 在新增或編輯腳本時，可以自訂其「顯示名稱」。
        - 列表上顯示的是我自訂的名稱。
        - 我可以隨時「重新命名」一個已經存在的腳本。

3.  **作為一名使用者，我希望能快速地找到我需要的腳本，特別是當列表很長的時候。**
    - **驗收條件**:
        - 提供一個搜尋框，我可以輸入關鍵字來即時篩選腳本列表。
        - 提供一組篩選按鈕，讓我可以只顯示特定類型的腳本 (如 Python, Batch)。

4.  **作為一名開發人員，我希望我的 Python 腳本能在其對應的虛擬環境中執行，以確保依賴套件的正確性。**
    - **驗收條件**:
        - 當執行一個 `.py` 腳本時，應用程式會自動從該腳本的目錄向上尋找 `venv` 或 `.venv` 資料夾。
        - 如果找到，它會使用該虛擬環境來執行腳本。
        - 如果找不到，它會提示我，並使用系統預設的 Python 來執行。

5.  **作為一名使用者，我想要能夠執行需要長時間運行的背景任務（例如一個網站伺服器），同時還能執行其他一次性的腳本。**
    - **驗收條件**:
        - 我可以將腳本設定為「背景常駐」或「一次性」。
        - 對於背景腳本，我能「啟動」和「停止」它。
        - 當有背景腳本在執行時，我仍然可以與應用程式的其他部分互動。

6.  **作為一名使用者，我需要看到我的腳本執行的輸出結果，以便了解執行狀況或進行偵錯。**
    - **驗收條件**:
        - 應用程式的主介面有一個專門的區域，用來即時顯示所選腳本的 console 輸出。
